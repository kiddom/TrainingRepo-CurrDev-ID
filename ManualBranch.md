# Fork It! (Modified) Exercise

Forking a repository creates a copy of it of the entire repo and a new branch. We're using curriculum-tools as a large folder at the moment. So if you copy it, you'll get everything in the repo, which is not what we want (yet). In some cases it might be practical to copy a skill and then tailor it for a specific grade band or content area. To do this, we can simply copy the file in the existing folder and edit it. This is not as clean as a fork if we wanted to merge the changes back in, but should work for most of our purposes for now. We can add additional repositories when the need arises - alignment skill for example ;-). 

1. copy this file (`ForkIt.md`) and rename it, then edit the practice block below.
2. **overwrite it** with whatever you'd like — a fun fact, a favorite subject, a quick sentence, anything.
3. **Commit** your change and open a pull request to post the file back to the repo.
